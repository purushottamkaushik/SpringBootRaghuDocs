                     Lombok API
                 [https://projectlombok.org/]

=> This is open source JAVA API.
=> This API is used to generate code, like
    set/get methods
    toString method override
    equals & hashCode
    constrcutors(default/param)
=> This process is done by using anntation.
    @Settet, @Getter, @ToString..etc

=>*** Lombok works at compilation time only. 
      Not required while running application.

=>*** In Developer machine, Lombok must be activated with IDE
      By using lombok jar only.

Lombok Annotations:-

1. @NoArgsConstructor : This annotation generated a default
      constrcutor before going to java compiler.
      While compile file, this annotation is removed
      and .class is genetaed.

2. @AllArgsConstructor : This annotation is used to generate
            one parameterized constructor with all variables
	    inside class.

3. @RequiredArgsConstructor + @NonNull
    To generated selected variables constrcutor.
    Here, we need to select variables using @NonNull


4. @Getter  : It will generate get methods for variables
   @Setter  : It will generate set methods for variables

5. @ToString : To override methods toString() use this

6. @EqualsAndHashCode: To generate equals() and hashCode()
                           methods use this.

7. @Data : It is a simplified annotation
      it is a combination of
           @Getter + @Setter + @ToString +
	   @EqualsAndHashCode + @RequiredArgsConstructor

Q) When Java compiler provides default constrcutor?
A) If there is no constrcutor given by programmer or lombok.

   If any constrcutor exist in code then java compiler
   never provides default constrcutor.


Q) How many constrcutor can we generated using Lombok at max?
A) Only 3 constrcutor at max.
   a. Default/zero param const
   b. All Param constrcutor
   c. Selected Params constrcutor.

   If you want any additional constrcutor then define code
   manually.
