Coding Part:-
 In Spring Boot application, read data from YML file using
 annotations : @Value / @ConfigurationProperties
 [same like properties concept only]


=> Duplicate keys are not allowed in YAMl File(.yml)
   If given you may see : DuplicateKeyException

=> Must provide one space before value, 
   else ScannerException

-------Syntaxes-------------------------------------
1. For a simple/primitive types
   prefix:
       key:<space>value

2. For List/Set/Array type use symbol -(dash) before value
   which indicates index.

   prefix:
     key:
     - value1
     - value2
     - value3

3. For Map type 
   prefix:
       key:
         mapKey:<space>mapValue

4. For Association Mapping(HAS-A) 

   prefix:
      hasAvariable:
           variable:<space>value

===FAQs========================================================
Q1) Can we use both properties and yml files in project?
A) YES. We can but not recomanded.
   If we define both files with key=val

   Spring container will search for key
      1st Inside Properties file , if not found then search
      2nd inside yaml file. Still not found then
         default value (if we load using @ConfigProps)
	 Exception     (if we try to load using @Value)

Q2) Can we delete properties file? Can we keep empty properties file?
A) Yes. Not a issue. We can delete/keep empty.
      @ConfigProps -- default value
      @Value       -- Exception

-----------------------------------------------------------------------
Q3)*** Can we write application without using properties/yml but still want to
     pass data? in key=val format!
A) Yes. 2 more concepts exist in Applications

     I. CLA (Command Line Args)  [--key=val]
    II. VM/System Args           [-Dkey=val]
